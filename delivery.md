## 1. GitHub Public Address
https://github.com/CarlosSallan/MTG-EDH-Deckbuilder

## 2. Design Decisions
- We developed a basic web application using the Django framework, adhering to the MVT (Model-View-Template) architecture. 
- The data model consists of 3 related entities (Deck, Card, DeckCard) to support the deckbuilder functionality, satisfying the complex relationship requirement. 
- The standard Django admin interface has been activated to add, modify, and delete instances.
- User authentication and registration have been successfully implemented. 
- We included the required `Dockerfile` and `docker-compose.yml` to run the application as a Docker container orchestration. 
- We adhered to the 12-factor guidelines where possible.
